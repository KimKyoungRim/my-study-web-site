1. 문서 구조

: 코드 입력시 부터 시작하는 문서 구조를 파악하고, 종이에도 직접 작성을 해서 이해도를 높힌다.

2. Tag?

  1. : '<' , '>' 로 묶어서 표현하는 명령어다.
  2. : 요소, 속성, 값 등으로 태그를 나눌 수 있다.
  3. : 태그는 '블록 요소' 와 '인라인 요소' 로 나뉜다.

- 블록 요소 (block element) -

  1. : 부모 요소의 전체 공간을 차지하여 '블록'을 만든다.
  2. : 하위요소를 포함하는 독립적인 형태의 상자를 의미. (ex. 레고를 연상해도 좋음 / but, 마크업에서는 블록이 아래로 쌓임 )
  3. : 블록 레벨 요소는 언제나 새로운 줄에서 시작하고, 좌우 양쪽으로 최대한 늘어나 가능한 모든 너비를 차지합니다.
  4. : 사이즈를 지정할 수 있다.
  5. : 하위요소로 하위 블록요소와 인라인요소 및 텍스트 요소를 포함할 수 있다.
  6. : 폭은 부모를 따라가고, 높이는 자식요소의 높이를 따른다.
  7. : 사이즈를 확인할려면 배경을 적용해 보는 것이 좋다.

- 인라인 요소 (inline element) -

  1. : ‘블록 상자 안의 일부’라는 의미로, 블록요소안에 포함된 하위 요소들로서 링크(a 요소), 이미지(img 요소),강조(em, strong 요소) 등이 해당됩니다.
  2. : 콘텐츠의 흐름을 끊지 않고, 요소를 구성하는 태그에 할당된 공간만 차지합니다.
  3. : 새로운 줄을 만들지 않으며 필요한 너비만 차지합니다.
  4. : 만약 인라인 요소의 사이즈를 지정하려면 CSS 속성중 'display' 속성을 이용해서 'block' 이나 'inline-block'으로 설정을 변경해주면 사이즈 지정이 가능하다.
  5. : 하위 요소로 블록요소를 포함할 수 없으며, 하위 인라인 요소와 텍스트 요소만 포함할 수 있다!
  6. : 사이즈는 폭과 높이 모두 자식요소의 폭과 높이만 따른다.
  7. : 쌓이지 않고 옆으로 붙는 성격을 가지고 있다.

3. Heading Tag ?

: 제목 작성을 위해 사용하는 태그로 'h1' 이후에 'h3' 등이 바로 오지 않도록 주의해야한다.

4. Paragraph Tag ?

  1. : 문단 작성을 위해 사용하는 태그로 'p' 를 사용함
  2. : 문단을 개행할 때 사용하는 태그는 'br' 이다.
  3. : 문단 요소 안에는 'a', 'img' 등과 같은 인라인 요소와 텍스트만 포함할 수 있으며, 블록 요소는 사용할 수 없습니다.

5. List Tag ?

  1. : UL (Unordered List) 순서가 중요하지 않는 컨텐츠에 사용하기 적절
  2. : OL (Ordered List) 순서가 중요한 컨텐츠에 사용하기 적절
  3. : DL (Definition List) 용어 정의 리스트를 생성할 때 사용

6. IMG Tag ?

: 웹 문서에 텍스트가 아닌 이미지 개체를 삽입할 때는 빈 요소인 img 요소를 이용해야 하며, img 요소의 경우 src 속성과 alt 속성은 필수 속성입니다. 특히 alt 속성은 이미지가 보이지 않는 환경에서 대체 텍스트를 제공하기 위한 목적으로 사용하며, 이미지와 동등한 정보를 제공해야 합니다.

: img src="이미지 명" alt="대체텍스트"
